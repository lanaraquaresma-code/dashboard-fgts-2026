🔄 Guia de Atualização do Dashboard

Pró-Transporte · FGTS 2026 · Como manter os dados sincronizados
1

Arquivo-base: BASE_FGTS_DASHBOARD.xlsx

Este arquivo é a única fonte de dados do dashboard. Mantenha a estrutura exatamente como está — mesmas colunas, mesma aba "Planilha1". Atualize os valores sempre que houver novas contratações, portarias ou mudanças na fila A Contratar.

2

Colunas para atualizar diariamente

A CONTRATAR AVANÇAR / NOVO PAC → quando entrar novo processo na fila
CONTRATADO EM 2026 AVANÇAR / NOVO PAC → quando um contrato for assinado
SELEÇÃO A PUBLICAR → quando uma portaria for publicada
VALOR EXECUTADO → Contratado Avancar + Contratado Novo PAC
SALDO → Limite − Valor Executado
COMPROMETIDO TOTAL → A Contratar total (sem Seleção)
SALDO REAL FUTURO → Saldo − Comprometido Total

3

Atualizar o dashboard via script Python

Com o arquivo Excel atualizado, execute o script:

python scripts/atualizar.py

O script lê os dados e gera um novo js/data.js automaticamente.

4

Estrutura do projeto

📁 FGTS_NOVO/ 📄 index.html ← abre no navegador 📁 css/ 📄 styles.css ← visual do dashboard 📁 js/ 📄 data.js ← gerado pelo Python 📄 helpers.js ← formatadores 📄 components.js ← renderizadores 📄 main.js ← inicialização 📁 data/ 📄 BASE_FGTS.xlsx ← planilha fonte 📁 scripts/ 📄 atualizar.py ← atualiza data.js
5

Problema de delay entre sistemas de contratação

Para resolver o problema de um sistema registrar a contratação e o outro não, vamos criar uma coluna de conciliação no arquivo base indicando a data de cada sistema e o status de sincronização. O dashboard vai exibir visualmente quando há divergência entre os sistemas — isso é o próximo passo que podemos construir juntos.
